# Architecture

TaskFlow follows an **OSS core + SaaS cloud** model with a monorepo structure.

## Repository Layout

```
todo/
├── core/                    # Open-source monorepo (MIT)
│   ├── apps/
│   │   ├── api/             # NestJS REST API
│   │   ├── web/             # Next.js web app
│   │   ├── mobile/          # Expo React Native app
│   │   └── desktop/         # Tauri desktop app
│   ├── packages/
│   │   └── shared/          # Shared types & utilities
│   ├── docker-compose.yml
│   ├── vitest.workspace.ts
│   └── package.json         # Bun workspace root
├── cloud/                   # SaaS cloud layer (proprietary)
├── docs/                    # This documentation site
└── package.json             # Root orchestrator
```

## API Layer

The API is a **NestJS** application using:

- **MongoDB + Mongoose** for data persistence
- **Clerk** for JWT-based authentication via `ClerkAuthGuard`
- **JSON:API** serialization via custom interceptors
- **Swagger/OpenAPI** for auto-generated API docs

### API Modules

| Module | Description |
|--------|-------------|
| `tasks` | CRUD + subtasks, reminders, recurrence, AI execution |
| `projects` | Project management with statuses, categories, tags |
| `comments` | Task comments |
| `goals` | Goal tracking |
| `connections` | External integrations |
| `history` | Activity/audit trail |
| `health` | Health check endpoint |

## Web Layer

The web app is **Next.js 15** with React 19, using:

- Clerk for authentication UI
- REST API calls to the NestJS backend
- Responsive design for desktop and mobile browsers

## Mobile Layer

**React Native** via **Expo**, sharing API client code with the web app.

## Desktop Layer

**Tauri** wrapping the React frontend for native desktop apps on macOS, Windows, and Linux.

## Data Flow

```
Client (Web/Mobile/Desktop)
    ↓ HTTP + Clerk JWT
NestJS API (ClerkAuthGuard)
    ↓ Mongoose
MongoDB
```
