# API Server

The TaskFlow API is a NestJS application providing a RESTful interface for all task management operations.

## Running

```bash
cd core
bun run dev:api
```

The API starts on `http://localhost:4000` by default.

## Key Features

- **JWT Authentication** via Clerk — all endpoints require a valid Bearer token
- **JSON:API Serialization** — responses follow JSON:API spec via custom serializers
- **Swagger Documentation** — auto-generated OpenAPI docs
- **Input Validation** — DTOs with `class-validator` decorators
- **Pagination** — all list endpoints support pagination

## Endpoints Overview

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/tasks` | Create a task |
| `GET` | `/tasks` | List tasks (filterable) |
| `GET` | `/tasks/:id` | Get a task |
| `PATCH` | `/tasks/:id` | Update a task |
| `DELETE` | `/tasks/:id` | Delete a task |
| `GET` | `/tasks/:id/subtasks` | Get subtasks |
| `POST` | `/projects` | Create a project |
| `GET` | `/projects` | List projects |
| `GET` | `/projects/:id` | Get a project |
| `PATCH` | `/projects/:id` | Update a project |
| `DELETE` | `/projects/:id` | Delete a project |
| `GET` | `/health` | Health check |

See the full [API Reference](/api-reference/tasks) for request/response details.
