# Configuration

All configuration is done via environment variables.

## API (`apps/api`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `MONGODB_URI` | Yes | — | MongoDB connection string |
| `PORT` | No | `4000` | API server port |
| `FRONTEND_URL` | Yes | — | Web app URL for CORS |
| `CLERK_SECRET_KEY` | Yes | — | Clerk secret key for JWT verification |

### Example `.env`

```env
MONGODB_URI=mongodb://localhost:27017/taskflow
PORT=4000
FRONTEND_URL=http://localhost:3000
CLERK_SECRET_KEY=sk_test_xxxxx
```

## Web App (`apps/web`)

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Yes | Clerk publishable key |
| `NEXT_PUBLIC_API_URL` | Yes | API server URL |

### Example `.env.local`

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxxxx
NEXT_PUBLIC_API_URL=http://localhost:4000
```

## Getting Clerk Keys

1. Create an account at [clerk.com](https://clerk.com)
2. Create a new application
3. Go to **API Keys** in the Clerk dashboard
4. Copy the **Publishable Key** and **Secret Key**

## MongoDB

TaskFlow uses MongoDB 7+. For local development, you can:

- Install MongoDB directly: `brew install mongodb-community`
- Use Docker: `docker run -d -p 27017:27017 mongo:7`
- Use the included `docker-compose.yml` which handles MongoDB automatically
