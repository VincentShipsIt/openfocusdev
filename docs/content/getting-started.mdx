# Getting Started

Get TaskFlow running locally in under 5 minutes.

## Prerequisites

- [Bun](https://bun.sh) (latest)
- [MongoDB](https://www.mongodb.com/) 7+ (or use Docker)
- [Clerk](https://clerk.com) account (for auth)

## Clone & Install

```bash
git clone https://github.com/shipshitdev/todo.git
cd todo/core
bun install
```

## Environment Variables

Create `.env` files in each app:

**`apps/api/.env`**
```env
MONGODB_URI=mongodb://localhost:27017/taskflow
PORT=4000
FRONTEND_URL=http://localhost:3000
CLERK_SECRET_KEY=sk_test_...
```

**`apps/web/.env.local`**
```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
NEXT_PUBLIC_API_URL=http://localhost:4000
```

## Run Development Servers

```bash
# Start both API and web
bun run dev

# Or individually
bun run dev:api    # API on :4000
bun run dev:web    # Web on :3000
bun run dev:mobile # Expo mobile
bun run dev:desktop # Tauri desktop
```

## Quick Docker Start

If you prefer Docker, from the `core/` directory:

```bash
docker compose up -d
```

This starts MongoDB, the API (port 4000), and the web app (port 3000). See the [Docker deployment guide](/guides/docker-deploy) for production setup.

## Verify

1. Open `http://localhost:3000` — you should see the TaskFlow login screen
2. Sign in via Clerk
3. Create your first project and task
