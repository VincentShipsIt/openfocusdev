# Clerk Auth Setup

TaskFlow uses [Clerk](https://clerk.com) for authentication across all platforms.

## Create a Clerk Application

1. Sign up at [clerk.com](https://clerk.com)
2. Create a new application
3. Choose your sign-in methods (email, Google, GitHub, etc.)
4. Copy your API keys from the dashboard

## Configure the API

The NestJS API uses `ClerkAuthGuard` to validate JWTs on every request.

Add to `apps/api/.env`:

```env
CLERK_SECRET_KEY=sk_test_xxxxx
```

The guard extracts the user ID from the Clerk JWT and makes it available via the `@CurrentUser()` decorator:

```typescript
@Get()
@UseGuards(ClerkAuthGuard)
findAll(@CurrentUser() user: { userId: string }) {
  return this.service.findAll(user.userId);
}
```

## Configure the Web App

Add to `apps/web/.env.local`:

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxxxx
```

The web app uses Clerk's React components for sign-in/sign-up UI and automatically attaches the JWT to API requests.

## Configure for Production

1. In Clerk dashboard, go to **API Keys**
2. Switch to **Production** instance
3. Use the production keys (`sk_live_`, `pk_live_`)
4. Configure your custom domain in Clerk settings
5. Set up webhooks if you need server-side user sync

## Supported Auth Methods

Configure in the Clerk dashboard:

- Email + password
- Magic links
- Google OAuth
- GitHub OAuth
- Apple Sign-In
- And more
