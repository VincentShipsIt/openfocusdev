### Decisions

- **Decision:** Added JSON:API deserialization in frontend API layer
  - **Context:** Backend uses JSON:API serializers that wrap responses in `{ data: [...] }` format with attributes nested
  - **Rationale:** Centralizing deserialization in api.ts keeps components clean and handles all endpoint responses consistently

- **Decision:** Reminders stored as array on Task rather than separate collection
  - **Context:** Reminders are tightly coupled to tasks
  - **Rationale:** Simpler queries, atomic updates, no orphaned records

- **Decision:** Background job for reminder notifications not implemented
  - **Context:** Plan mentioned reminders require background jobs
  - **Rationale:** Core CRUD is in place; notification triggering requires scheduler infrastructure (e.g., @nestjs/schedule) which is out of scope for this session

### Mistakes and fixes

- **Mistake:** Date state in task-form.tsx used `slice(0, 16)` getting datetime, then concatenated with time causing invalid `YYYY-MM-DDTHH:mmTHH:mm`
- **Fix:** Changed to `slice(0, 10)` for date and `slice(11, 16)` for time from ISO string
- **Prevention:** Use consistent ISO string slicing for date/time extraction

- **Mistake:** JSON:API responses not being deserialized, causing `.filter is not a function` errors
- **Fix:** Added `deserializeJsonApi` function to transform JSON:API format back to plain objects
- **Prevention:** Always check API response format when integrating with serialized backends

- **Mistake:** Task and Project serializers missing new attributes
- **Fix:** Added `parentTaskId`, `recurrence`, `reminders` to TaskSerializer and `isFavorite` to ProjectSerializer
- **Prevention:** Update serializers when adding new schema fields

### Decisions

- **Decision:** Separate connections collection instead of embedding in tasks
  - **Context:** Task connections could be embedded in tasks or separate collection
  - **Rationale:** Separate collection allows efficient cycle detection traversal and prevents document bloat

- **Decision:** Two connection types: dependency (blocking) and sequence (non-blocking)
  - **Context:** User requested task connections
  - **Rationale:** Dependencies block downstream tasks; sequences show flow without blocking

- **Decision:** AI execution placeholder (marks complete)
  - **Context:** Full AI integration requires external service
  - **Rationale:** Infrastructure in place for future AI integration; currently simulates completion

### Mistakes and fixes

- **Mistake:** TypeScript errors with React Flow generic types (useNodesState/useEdgesState)
- **Fix:** Added explicit generic types `useNodesState<TaskNode>([])` and `useEdgesState<TaskEdge>([])`
- **Prevention:** Always provide explicit generic types when using React Flow hooks

- **Mistake:** parseISO called on `Date | string` type
- **Fix:** Added type check: `typeof task.dueDate === 'string' ? parseISO(task.dueDate) : task.dueDate`
- **Prevention:** Handle union types explicitly in date parsing
